Explicação do Projeto
Este é um projeto Spring Boot + Jersey para uma API REST de gerenciamento de usuários com interface web.
---
pom.xml (Configuração Maven)
Linha	Explicação
1-2	Declaração XML do projeto Maven
3-5	Identificação do projeto: grupo br.edu.utfpr.td.tsi, artifact exemplo-webapi, versão 0.0.1-SNAPSHOT
6-11	Herança do Spring Boot Parent v3.2.3 (fornece dependências e plugins)
13-19	Properties: UTF-8, Java 17
21-36	Dependências: spring-boot-starter, spring-boot-starter-web, spring-boot-starter-jersey
38-45	Plugin Spring Boot para empacotamento
---
Main.java (Ponto de entrada)
Linha	Explicação
1	Declaração do package
3-4	Imports do Spring Boot
6	@SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan
9-11	Método main que inicia a aplicação Spring
---
JerseyConfiguration.java (Configuração REST)
Linha	Explicação
1	Package
3-10	Imports do Jersey e Spring
13	@Component = bean gerenciado pelo Spring
14	@ApplicationPath("/exemplo-webapi") = prefixo das URLs da API
15-21	Registra o endpoint e o filtro CORS
23-34	CorsFilter: adiciona headers HTTP para permitir requisições de outras origens (CORS)
---
UserEndpoint.java (API REST)
Linha	Explicação
1	Package
3-5	Imports de collections e JAX-RS
15	@Path("usuarios") = URL base /exemplo-webapi/usuarios
18	Lista estática em memória (simula banco de dados)
20-26	POST /usuarios: recebe JSON de User, adiciona à lista, retorna 201 Created
28-32	GET /usuarios: retorna lista de usuários em JSON
---
User.java (Model)
Linha	Explicação
1	Package
5-7	Atributos: nome, email, idade
9-10	Construtor vazio (necessário para deserialização JSON)
12-16	Construtor com parâmetros
18-40	Getters e setters para cada atributo
---
### **HTMLs** (Frontend)
**index.html** - Página inicial com menu
- Linhas 7-55: CSS com gradiente roxo, layout centralizado
- Linhas 62-64: Links para cadastro e listagem
**cadastro.html** - Formulário de cadastro
- Linhas 97-112: Form com campos (nome, email, idade)
- Linhas 116-153: JavaScript que faz `POST /exemplo-webapi/usuarios` e redireciona para listagem
**listagem.html** - Tabela de usuários
- Linhas 102-131: JavaScript que faz `GET /exemplo-webapi/usuarios` e popula a tabela
---
Fluxo da aplicação
1. Usuário acessa index.html → clica "Cadastrar Usuário"
2. Preenche formulário → JavaScript envia POST para API
3. API adiciona usuário na lista em memória
4. Redireciona para listagem → JavaScript faz GET e exibe tabela
